# Data Engineering Journey

Learning data engineering in public — from messy datasets to production-grade pipelines. Each lesson is a complete, self-contained notebook documenting what I'm learning, how I'm thinking, and the mistakes I'm catching along the way.

## Why this repo exists

I'm transitioning into a career in **Data Engineering** (with some AI Engineering on the side). Rather than learn quietly and hope it sticks, I'm working through this publicly partly to hold myself accountable, partly because explaining what I'm learning is the best way to actually learn it.

If you're earlier in this journey than me, hopefully something in here helps you. If you're further along, I'd love feedback.

## Roadmap

I'm working through data engineering in three broad phases:

**Phase 1 — Foundations** *(currently here)*
SQL, Python, pandas, GCP, dbt, Airflow, Docker. Building muscle on data cleaning, transformation, and pipeline basics.

**Phase 2 — Real projects**
End-to-end pipelines on real datasets (starting with the Olist e-commerce dataset). GCP, BigQuery, dbt, Airflow.

**Phase 3 — Scale and specialization**
Streaming, Spark at scale, ML/AI engineering branches, production-grade systems.
